Fase 1

  • Data inlezen
  • Code sturen

We lezen alle data in in een database, waar we vervolgens verder mee kunnen werken. Vervolgens wordt voor elke leerling een unieke code gegenereerd waarvan de hash wordt opgeslagen in de database, tegelijkertijd wordt de code naar de leerling gestuurd zodat deze nergens hoeft worden opgeslagen. Indien nodig kan de code altijd her gegeneerd worden.

Database

De database is een MySQL database, deze wordt gehost op de server van de school. De database is beveiligd met een wachtwoord en is alleen toegankelijk voor de ontwikkelaar.


Inlezen

Het inlezen van de data moet handmatig worden gedaan, de ontwikkelaar zal hier bij helpen.

Code's genereren

De code wordt gegenereerd door een algoritme. De code wordt gegenereerd door een random string van 6 karakters te genereren, deze wordt gelijk naar de leerling gemaild. Hierna wordt deze string vervolgens ge-hashd met een SHA256 algoritme en opgeslagen in de database. Door deze methode is het niet mogelijk om de code te achterhalen.



Code hergeneren

Indien nodig kan de code altijd her gegenereerd worden. Dit kan handig zijn als de code per ongeluk is verwijderd of als de code niet is aangekomen.
